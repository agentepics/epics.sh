Epics are not one-shot prompts. They are durable workflow packages for coding
agents.

They are aimed at jobs that need continuity:

- multi-session implementation work
- research with sources and conclusions
- release and operations flows
- work that has to survive interruption or handoff

The package model stays host-neutral. The `epics` CLI is the canonical behavior
surface, while supported hosts are expected to uphold the same autonomy
contract instead of exposing downgraded compatibility tiers.
